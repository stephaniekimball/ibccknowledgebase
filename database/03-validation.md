# Caller Validation Procedure

## Call Greeting
"Thank you for calling [Company Name]. This is [Your Name]. How can I assist you today?"

## Call Closing
"Thank you for calling [Company Name]. Have a great day!"

## Core Rules
1. Verify identity before discussing account details.
2. Always ask for the account holder or authorized user's information.
3. Follow confidentiality protocols.

## Inbound Account Owner Validation (4 Variables)
1. Full Name (as per account)
2. Date of Birth
3. Account Number
4. Last Four Digits of Social Security Number (if applicable)

## Inbound Authorized User Validation
- Authorized users must provide their name and the account owner's consent.

## Outbound Validation
- If contacting a client, confirm their identity using the same variables as inbound validation.

## Failed Verification
- If verification fails, inform the caller politely:
  "I’m sorry, but I am unable to verify your identity at this time."

## Internal Transfers
- If the caller needs to be transferred, confirm their details again before the transfer.
- Use:
  "I will transfer you to [Department]. Please hold for a moment."

## Steps to Validate
1. Greet the caller.
2. Ask for their account information.
3. Validate using the core rules.
4. Confirm or deny access based on verification.
5. Close the call appropriately.

## Third Party Callers Approved Language
- For third-party callers:
  "I’m sorry, but I cannot disclose account details without the account holder present."
